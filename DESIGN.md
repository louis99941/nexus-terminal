# 星枢终端（Nexus Terminal）设计文档

> 本文档记录项目的核心架构决策、技术选型理由和已知限制，供后续开发者理解"为什么这样做"。

---

## 设计概述

### 目标

星枢终端是一个基于 Web 的远程连接管理平台，核心目标是：

1. **协议统一**：在单一浏览器界面中管理 SSH、SFTP、RDP、VNC、Telnet 五种远程协议
2. **轻量部署**：三个 Docker 容器覆盖全部功能，无需外部数据库或消息队列
3. **安全优先**：所有连接凭据加密存储，容器以非 root 运行，完整审计日志
4. **高度可定制**：终端主题、布局、背景动效、键盘映射均可用户自定义

### 非目标

- 不做多租户 SaaS 平台（当前设计面向单用户/小团队自部署）
- 不做浏览器端 SSH 代理（SSH 连接由后端 Node.js 进程建立，非 WebSocket 直连服务器）
- 不做实时协作编辑（终端会话是独占式的，不支持多人同时操作同一终端）

### 数据流

**SSH/Telnet 连接**：浏览器 → WebSocket → Backend（Node.js SSH2 客户端）→ 目标服务器
**SFTP 文件操作**：浏览器 → HTTP API → Backend（ssh2 SFTP 子系统）→ 目标服务器
**RDP/VNC 连接**：浏览器 → WebSocket → Remote Gateway（guacamole-lite）→ Guacd（内嵌）→ 目标服务器

## 核心设计决策

### 1. SQLite 而非 PostgreSQL

**决策**：使用 SQLite 作为唯一数据库。

**理由**：
- 单用户/小团队场景下，SQLite 的性能完全足够（WAL 模式支持并发读写）
- 零配置部署——不需要额外的数据库容器，Docker Compose 三容器即可运行
- 数据文件（`./data/nexus.db`）可直接备份/迁移，运维成本极低
- 后端通过进程内缓存（settings 5 分钟 TTL、connections 2 分钟 TTL）减少高频 SQL 查询

**代价**：
- 不支持多实例水平扩展（SQLite 文件锁在多进程写入时会成为瓶颈）
- 缺少高级查询能力（如全文搜索、窗口函数），审计日志查询依赖内存过滤

**放弃的替代方案**：
- PostgreSQL：部署复杂度增加一个容器，且本项目不需要关系型数据库的高级特性
- Redis：仅适合缓存，不适合作为主存储

### 2. Guacamole 内嵌而非独立容器

**决策**：将 Guacd 进程内嵌于 remote-gateway 容器，而非作为独立容器部署。

**理由**：
- 减少容器数量（从 4 个精简为 3 个），降低运维复杂度
- Guacd 与 guacamole-lite 通过 localhost:4822 通信，延迟最低
- ARM64 架构下无需单独处理 guacd 镜像兼容性

**代价**：
- Guacd 进程崩溃会影响整个 remote-gateway 容器
- 无法独立扩展 Guacd 实例（但单实例已足够应对并发 RDP/VNC 连接）

**替代方案**：
- 独立 guacd 容器：部署更复杂，且 Docker 网络增加了 ~0.1ms 延迟，收益不明显

### 3. WebSocket 多路复用

**决策**：支持单 WebSocket 连接承载多个 SSH 会话（默认关闭，通过 `ENABLE_MULTIPLEX` 启用）。

**理由**：
- 浏览器对同一域名的并发 WebSocket 连接数有限制（HTTP/1.1 下约 6 个）
- 多路复用模式下，多个终端标签共享一个物理连接，减少浏览器资源消耗
- 服务端通过 session ID 区分不同会话的消息路由

**代价**：
- 实现复杂度增加（消息分帧、会话隔离、错误传播）
- 关闭时回退到传统的每会话一个连接模式，保证兼容性

### 4. 终端输入延迟优化策略

**决策**：采用智能批处理 + TCP_NODELAY 的组合策略。

**理由**：
- SSH 终端的输入延迟是用户体验的核心指标（原始延迟 72-232ms，优化后 <3ms）
- 小数据包（<64 字节）直接写入 SSH 流，避免批处理开销
- 大数据块（如粘贴多行文本）使用 16ms 微批处理，合并多次 `data` 事件
- 后端 SSH 连接设置 `TCP_NODELAY = true`，禁用 Nagle 算法

**权衡**：
- 高频小包直写增加了网络包数量，但对现代网络环境影响可忽略
- 批处理窗口 16ms 在极端情况下可能引入感知延迟，但实测用户无感知

### 5. AI 多 Provider 架构

**决策**：支持 OpenAI、Claude、本地模型等多种 AI Provider，通过统一接口适配。

**理由**：
- 用户可能有不同 Provider 的 API Key，或需要使用本地部署的模型
- 后端 `ai-ops` 模块通过 `AI_PROVIDER_DEFAULTS` 常量管理各 Provider 的默认配置
- 前端 AI 设置页面支持下拉选择 Provider，自动调整 Endpoint 和模型参数

**实现**：
- 统一的 prompt 模板系统，不同 Provider 共享相同的消息格式
- 429 错误自动重试 + 降级逻辑
- AI 客户端使用 LRU 缓存（上限 16），避免重复创建连接

### 6. 事件驱动架构

**决策**：使用增强版事件总线（`event.service.ts`）实现模块间松耦合通信。

**理由**：
- 50+ 事件类型覆盖认证、连接、文件操作、审计等全流程
- 类型安全的事件 payload（`EventPayloadMap`），编译时检查事件参数
- 错误隔离：单个事件监听器的异常不会影响其他监听器
- 中间件链支持：日志记录、事件持久化等横切关注点通过中间件实现
- 领域命名空间：按模块划分事件（`ssh:*`、`sftp:*`、`auth:*`），避免命名冲突

## 安全设计

### 威胁模型

| 威胁 | 影响 | 缓解措施 |
|------|------|----------|
| 凭据泄露 | 远程服务器被未授权访问 | AES-256-GCM 加密存储，密钥轮换支持 |
| 命令注入 | 通过 SFTP/Docker 接口执行任意命令 | 统一 `shellEscape()` 工具函数 + 白名单校验 |
| SSRF 攻击 | 通过 URL 参数访问内网资源 | `validateUrlNotPrivate()` 域名精确匹配 |
| XSS 攻击 | 窃取用户会话或执行恶意操作 | CSP 安全头 + Helmet 中间件 |
| 暴力破解 | 暴力猜测密码 | bcrypt 12 轮 + IP 黑名单 + 速率限制 |
| WebSocket 劫持 | 跨站 WebSocket 劫持（CSWSH） | Origin 白名单校验 |
| 路径穿越 | 通过文件操作访问任意路径 | `VALID_TABLE_NAME` 正则白名单 + `VALID_SUSPEND_ID` 校验 |

### 安全措施层次

```
┌─────────────────────────────────────────┐
│  Layer 5: 容器安全                        │
│  非 root 运行 + 资源限制 + 日志轮转        │
├─────────────────────────────────────────┤
│  Layer 4: 网络安全                        │
│  CSP + CORS + Origin 白名单 + HTTPS       │
├─────────────────────────────────────────┤
│  Layer 3: 认证授权                        │
│  Session + 2FA + Passkey + IP 白名单      │
├─────────────────────────────────────────┤
│  Layer 2: 输入验证                        │
│  Zod schema + shellEscape + SSRF 防护     │
├─────────────────────────────────────────┤
│  Layer 1: 数据保护                        │
│  AES-256-GCM 加密 + 密钥轮换 + 日志脱敏   │
└─────────────────────────────────────────┘
```

## 已知限制

1. **SQLite 写入瓶颈**：多实例部署时 SQLite 文件锁会成为瓶颈，升级到 PostgreSQL 需要较大改造
2. **单用户设计**：当前没有用户注册/多租户机制，所有配置共享同一个 SQLite 文件
3. **终端会话独占**：不支持多人同时操作同一终端（如结对编程），需要重新设计会话模型
4. **Guacd 内嵌限制**：Guacd 崩溃会拖垮整个 remote-gateway，无法独立重启
5. **WebGL 渲染兼容性**：部分旧浏览器或虚拟机可能不支持 WebGL，需降级到 Canvas 渲染
6. **AI Provider 依赖**：AI 功能依赖外部 API（OpenAI/Claude），网络不可用时降级为本地提示
